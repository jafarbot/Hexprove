## What Was Implemented

### 1. Blog Scroll Depth Tracking

**Purpose:** Measure content engagement quality

**Implementation:**
- Created `lib/useScrollDepth.ts` hook
- Tracks scroll milestones: 25%, 50%, 75%, 100%
- Throttled scroll events for performance
- Integrated into `components/blog/BlogPost.tsx`

**Data Captured:**
```javascript

  page_path: '/blog/truebit-26m-logic-bug',

  time_on_page: 180, // seconds
}
```

**Business Value:**
- **Marketing:** Validates content quality and SEO investment
- **Sales:** Identifies engaged readers (warmer leads)
- **Engineering:** Measures page performance impact on engagement

## How to Use

### For Blog Posts (Automatic)
Scroll depth tracking is automatic for all blog posts. No additional code needed.

### For Homepage Sections (Automatic)
Section view tracking is automatic for all main sections. No additional code needed.

### For Outbound Links (Manual)
Use the `HoverText` component with tracking props:

```tsx
<HoverText 
  text="LinkedIn" 
  href="https://www.linkedin.com/in/sinousmonov/" 
  className="..."
  trackLocation="contact_footer"
  onTrack={trackOutboundLink}
/>
```

Or use the `TrackableLink` component:

```tsx
<TrackableLink 
  href="https://uniswap.org"
  location="hero_experience_section"
>
  Uniswap
</TrackableLink>
```

## Performance Considerations

### Scroll Depth Tracking
- **Throttled:** Uses `requestAnimationFrame` to prevent performance issues
- **Memory:** Tracks milestones in a Set to prevent duplicate events
- **Impact:** Negligible (<1ms per scroll event)

### Section View Tracking
- **Native API:** Uses Intersection Observer (browser-optimized)
- **One-time:** Each section tracked only once per page load
- **Impact:** Negligible (browser handles all calculations)

### Outbound Link Tracking
- **Synchronous:** Fires immediately on click
- **Non-blocking:** Uses `sendBeacon` API for reliability
- **Impact:** <1ms per click

## Support & Troubleshooting

### Events Not Showing in Console?
- Check `.env.local` has `NEXT_PUBLIC_ENABLE_ANALYTICS=true`
- Restart dev server (`npm run dev`)

### Events Not in Umami Dashboard?
- Check Umami website ID is correct in `.env.local`
- Verify Umami script is loading (check Network tab)
- Wait 1-2 minutes for events to appear (not real-time)

### Scroll Depth Not Firing?
- Check if page is tall enough (must be scrollable)
- Open console and scroll slowly to see events
- Verify `useScrollDepth` hook is called in component

### Section Views Not Firing?
- Check if section is visible in viewport
- Verify `useSectionTracking` hook is called
- Ensure ref is properly attached to section element
