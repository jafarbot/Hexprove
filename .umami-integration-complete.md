## Your Next Steps (5 minutes)

### Quick Cloud Setup:

1. **Sign up:** https://cloud.umami.is
   - Plan: Hobby ($9/mo)
   - Email: team@hexprove.com (or your email)

2. **Add website:**
   - Name: Hexprove
   - Domain: hexprove.com

3. **Get Website ID:**
   - Settings → Websites → Hexprove → Edit
   - Copy the Website ID

4. **Add to `.env.local`:**
   ```bash
   NEXT_PUBLIC_UMAMI_WEBSITE_ID=paste-id-here
   NEXT_PUBLIC_UMAMI_URL=https://cloud.umami.is/script.js
   ```

5. **Restart dev server:**
   ```bash
   # In terminal running npm run dev, press Ctrl+C
   npm run dev
   ```

6. **Test:**
   - Visit http://localhost:3003
   - Go to Umami dashboard → Realtime
   - You should see yourself!

7. **Deploy to Vercel:**
   - Add same env vars to Vercel project settings
   - Push to main branch
   - Analytics will be live!

## FAQ

### Q: Do I need both Umami and BigQuery?

**A:** No. Umami is enough for most needs. BigQuery is only if you want to run complex custom queries on raw data.

**My recommendation:** Just use Umami. Your BigQuery code will stay dormant (it gracefully falls back if not configured).

### Q: Can I test before deploying to production?

**A:** Yes! Add to `.env.local`:
```bash
NEXT_PUBLIC_ENABLE_ANALYTICS=true
```

Then analytics will work on localhost:3003 and you can test in Umami dashboard.

### Q: What if I want to switch later?

**A:** Easy. Just change the Website ID to point to a different Umami instance (cloud → self-hosted or vice versa).

### Q: Is Umami GDPR compliant?

**A:** Yes! No cookies, no personal data, privacy-focused by design. You don't even need a cookie banner.
