## Recommended Analytics Stack (Google-Free Alternatives)

### Option 1: Plausible Analytics (RECOMMENDED)

**Best for:** Privacy-focused, simple analytics without BigQuery complexity

**Pros:**
- Privacy-focused (no cookies, GDPR compliant)
- Dead simple setup (one script tag)
- Beautiful, intuitive dashboard
- €9/month (10k pageviews) - affordable
- No cookie banners needed
- Tracks all the basics: pages, referrers, goals

**Cons:**
- Less granular than BigQuery custom events
- Can't query raw data like BigQuery
- Limited event properties

**Implementation:**
```tsx
// 1. Add to app/layout.tsx (in <head>)
<Script defer data-domain="hexprove.com" src="https://plausible.io/js/script.js" />

// 2. Track custom events (CTA clicks, form submits)
<Script defer data-domain="hexprove.com" src="https://plausible.io/js/script.manual.js" />

// In components:
onClick={() => {
  window.plausible('CTA Click', { props: { location: 'hero', text: 'Book a Call' } });
  trackCtaClick('hero', 'Book a Call', '/#contact');
}}
```

**Cost:** €9-29/month depending on traffic

**Setup time:** 15 minutes

### Option 3: PostHog (RECOMMENDED - If You Want Product Analytics)

**Best for:** Product analytics, session replay, feature flags

**Pros:**
- Open source
- Session replay (see user journeys)
- Funnel analysis
- Feature flags built-in
- Self-hosted or cloud
- Generous free tier (1M events/mo)
- Can replace BigQuery for product analytics

**Cons:**
- More complex than Plausible/Umami
- Heavier script (~30KB)
- Requires more setup

**Implementation:**
```tsx
// app/providers.tsx
'use client'
import posthog from 'posthog-js'
import { PostHogProvider } from 'posthog-js/react'

if (typeof window !== 'undefined') {
  posthog.init(process.env.NEXT_PUBLIC_POSTHOG_KEY!, {
    api_host: 'https://app.posthog.com',
    capture_pageview: false // We do manual tracking
  })
}

// Track events
posthog.capture('cta_click', { location: 'hero', text: 'Book a Call' })
```

**Cost:** Free up to 1M events/month, then $0.00031/event

**Setup time:** 1-2 hours

## BigQuery Setup (For Later)

When you're ready for deep analytics:

### Step 1: Create BigQuery Project

```bash
# 1. Go to console.cloud.google.com
# 2. Create project: "hexprove-analytics"
# 3. Enable BigQuery API
# 4. Create dataset: "website_events"
# 5. Create table: "events" with schema:
```

**Schema:**
```sql
CREATE TABLE website_events.events (
  eventName STRING,
  timestamp TIMESTAMP,
  sessionId STRING,
  referrer STRING,
  userAgent STRING,
  url STRING,
  pagePath STRING,
  deviceType STRING,
  utmParams JSON,
  timeOnSite INT64,
  data JSON
);
```

### Step 2: Generate Credentials

```bash
# 1. Go to IAM & Admin → Service Accounts
# 2. Create service account: "hexprove-analytics"
# 3. Grant role: "BigQuery Data Editor"
# 4. Create key (JSON)
# 5. Download credentials.json
```

### Step 3: Add to .env.local

```bash
BIGQUERY_PROJECT_ID=hexprove-analytics
BIGQUERY_DATASET=website_events
BIGQUERY_CREDENTIALS='{"type":"service_account","project_id":"hexprove-analytics",...}'
```

### Step 4: Install Package

```bash
npm install @google-cloud/bigquery
```

### Step 5: Test

```bash
# Visit site, trigger events
# Check BigQuery console for data
```
