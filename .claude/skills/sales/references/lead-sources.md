## Lead Source Directory

### Funding Announcements

**Crunchbase**
- URL: crunchbase.com
- Search: "blockchain" OR "crypto" OR "web3" OR "defi"
- Filter: Funding date (last 30 days), Stage (Seed to Series B)
- Signal: Recently funded = hiring/scaling = need QA

**The Block Research**
- URL: theblock.co/data/deals
- Filter: By round size, category
- Signal: Funding announcements with details

**Crypto Twitter**
- Search: "raised" AND ("seed" OR "series a" OR "funding") AND ("crypto" OR "web3" OR "defi")

- Companies posting QA roles (they need QA)

## Lead Research Process

### Step 1: Basic Qualification

Before deep research, quick check:

| Criterion | How to Check | Pass? |
|-----------|--------------|-------|
| Is it crypto/web3? | Website, product | ✓/✗ |
| Team size 5+? | LinkedIn, website | ✓/✗ |
| Has funding? | Crunchbase, announcements | ✓/✗ |
| Consumer product? | Product description | ✓/✗ |
| No large QA team? | LinkedIn search | ✓/✗ |

If any ✗ on critical items, move on.

### Step 2: Deep Research

For qualified leads, gather:

**Company Info**
- Company name
- Website URL
- One-line description
- Team size (LinkedIn count)
- Location/HQ

**Product Details**
- What they build (wallet, DEX, marketplace, etc.)
- Platforms (web, iOS, Android, extension)
- User base size (if public)
- Key features
- Recent launches

**Funding & Stage**
- Last round (amount, date)
- Total raised
- Investors
- Stage (seed, A, B)

**Team & Contacts**
- Founder(s) names, LinkedIn, Twitter
- CTO / VP Eng name, LinkedIn, Twitter
- Any QA hires visible?
- Email pattern (try Hunter.io, or guess firstname@domain)

**Pain Signals**
- Job postings (QA roles? Engineering roles?)
- Twitter complaints about bugs
- Recent product issues
- Rapid scaling signs

**Personalization Hooks**
- Recent news/announcements
- Founder's background (shared connections?)
- Product features to mention
- Competitors they're facing

### Step 3: Document in Pipeline

Create entry with:
```
Company: [Name]
Website: [URL]
Product: [Description]
Team Size: [X]
Funding: [Stage, Amount, Date]
Contact: [Name, Title, Email, LinkedIn, Twitter]
Pain Signals: [List]
Personalization: [Hooks for outreach]
Status: Lead Identified
Priority: [High/Medium/Low]
Notes: [Anything relevant]
```

## Lead Scoring Framework

Score leads 1-10 based on:

| Factor | Points | Criteria |
|--------|--------|----------|
| Funding recency | 0-2 | 2 = last 30 days, 1 = last 90 days, 0 = older |
| Team size | 0-2 | 2 = 15+, 1 = 5-14, 0 = <5 |
| Product fit | 0-2 | 2 = perfect fit (wallet, DEX), 1 = good fit, 0 = marginal |
| Pain signal | 0-2 | 2 = strong signal (job posting, complaint), 1 = some signal, 0 = none |
| Contact access | 0-2 | 2 = have email + warm intro, 1 = have email, 0 = no contact |

**Priority:**
- 8-10: High priority, outreach immediately
- 5-7: Medium priority, batch outreach
- 1-4: Low priority, nurture/monitor

**Monday (30 min):**
- Check Crunchbase for new funding (last 7 days)

- Search job boards for QA postings
- Check LinkedIn for new crypto QA roles

## Contact Finding Tips

**Email Patterns to Try:**
- firstname@company.com (most common)
- first.last@company.com
- firstinitiallastname@company.com
- first@company.com

**Tools:**
- Hunter.io — Email finder
- Apollo.io — Contact database
- LinkedIn Sales Navigator — Best for B2B
- Twitter DMs — Often open for founders

**If Can't Find Email:**
- LinkedIn connection request with note
- Twitter DM if DMs are open
- Engage with their content first, then DM
- Find mutual connection for intro
